# 中文学术论文写作规范（Chinese Academic Writing Conventions）

> 中文学术论文特有的写作规范，涵盖人称、语态、数字、图表引用、中英混排等方面。

---

## 一、第一人称用法

中文学术论文中的"第一人称"比英文论文更复杂，因为中文有"本文""笔者""我们""本研究"等多种选择，各有适用场景和禁忌。

### 各人称的适用场景

| 表达方式 | 适用场景 | 示例 | 禁忌 |
|---------|---------|------|------|
| **本文** | 指代论文本身（最常用，最安全） | "本文提出了一种基于{{方法}}的{{方案}}。" | 不能说"本文认为"，因为论文不能"认为" |
| **本研究** | 指代研究活动/项目 | "本研究采用{{方法}}对{{对象}}进行分析。" | 范围比"本文"更宽，不适合指代论文写作行为 |
| **我们** | 指代作者团队（建议多人合著使用） | "我们设计了{{实验}}来验证{{假设}}。" | 独著论文用"我们"略有不自然 |
| **笔者** | 指代论文作者（独著更适用） | "笔者在{{场景}}中观察到……" | 过度使用会显得古典或生硬 |
| **本工作** | 指代研究工作的成果 | "本工作首次在{{领域}}中实现了{{目标}}。" | 较新的用法，部分导师可能不认同 |

### 动词搭配规则

```
✅ "本文提出/设计/构建/验证了……"     （论文作为主体，搭配认知/行为动词）
❌ "本文认为/发现/观察到……"          （论文没有思想，不能搭配认知动词）

✅ "我们认为/发现/观察到……"          （作者可以认知）
✅ "实验结果表明……"                  （最客观的绕过方式）
```

### 使用建议

- **默认首选"本文"**：最安全、最符合国内大多数学术期刊的惯例
- **用"我们"时注意一致性**：不要在同一章节中混用"本文""我们""笔者"
- **能用客观表达就不用第一人称**："实验结果表明"优于"我们发现实验结果表明"
- **避免过度使用"笔者"**：尤其在理工科论文中，"笔者"可能显得过于人文气息

## 二、被动与主动语态

中文学术论文对被动/主动的选择与英文不同——中文被动句有天然的"受损"含义，过度使用会让文字生硬。

### 语态选择原则

| 原则 | 说明 | 示例 |
|------|------|------|
| 优先主动 | 中文习惯用主动语态 | ✅ "{{系统}}通过{{模块A}}实现{{功能}}" |
| 慎用"被"字句 | "被"在中文中常含消极意味 | ❌ "数据被收集" → ✅ "收集数据" / "数据采集过程" |
| 可用"由/经/通过"替代"被" | 无标记被动 | "{{方案}}由{{方法A}}和{{方法B}}两部分组成" |
| 强调过程用"对……进行" | 形式主动语义被动 | "对{{对象}}进行了{{处理}}" |
| 英文论文 | 英文更倾向被动语态 | "The experiment was conducted..." |

### 常见替换对照

| 原始表达（生硬） | 修改后（自然） |
|----------------|---------------|
| "数据被采集后，被送入处理模块" | "采集后的数据进入处理模块" |
| "实验结果被展示在表中" | "实验结果见表{{X}}" |
| "该系统被设计为……" | "该系统采用……设计" |
| "问题被解决" | "解决了该问题" / "该问题得以解决" |

## 三、时间表述规范

中文学术论文中对时间的表述需区分"模糊时间"和"精确时间"。

### 时间表述建议

| 表述类型 | 适用场景 | 示例 | 注意事项 |
|---------|---------|------|---------|
| "近年来" | 描述近期趋势 | "近年来，{{领域}}发展迅速。" | 不要频繁重复使用 |
| "目前/当前" | 描述现状 | "目前，该领域的研究主要聚焦于……" | 在引言和文献综述中使用 |
| "近五年来" | 更精确的近期 | "近五年来，关于{{主题}}的论文数量增长了{{N}}倍。" | 比"近年来"更具体 |
| 具体年份 | 引用具体事件 | "自2020年起，{{法规/标准}}要求……" | 用于需要精确时间锚的时刻 |
| "截至{{时间}}" | 数据截止时间 | "截至2025年6月，共采集{{N}}条数据。" | 实验/调查类论文必用 |

### 避免的表达

```
❌ "最近" — 过于口语化
❌ "现在" — 过于口语化
❌ "前不久" — 模糊且口语化
❌ "长期以来" — 需要数据支撑，否则空洞
```

## 四、数字与单位规范

中文论文中数字和单位的使用有严格规范，核心规则：**汉字与阿拉伯数字各有其适用范围**。

### 数字使用规则

| 规则 | 用阿拉伯数字 | 用汉字数字 |
|------|-----------|-----------|
| 计量/统计数值 | ✅ "平均响应时间为35 ms" | — |
| 百分比 | ✅ "准确率达98.5%" | — |
| 编号/代码 | ✅ "第4章""图3""表2" | — |
| 年份/日期 | ✅ "2025年6月" | — |
| 约数（一位数） | — | ✅ "三种方法""五类问题" |
| 成语/固定搭配 | — | ✅ "一言以蔽之""三方面" |
| 作为词素 | — | ✅ "二极管""四边形" |
| 段首避免用阿拉伯数字开头 | — | ✅ "二十所高校参与了调查" |

### 单位使用规则

```
✅ 使用国际标准单位符号（SI）：ms, MB, GHz, μm
✅ 数字与单位之间留半角空格："35 ms"而非"35ms"
✅ 复合单位用"/"："km/h"
❌ 不要用中文单位全称替代标准符号："35毫秒" → "35 ms"
✅ 首次出现非标准简写时给出全称："平均无故障时间（MTBF）"
```

## 五、图表引用规范

图表是学术论文的重要辅助，但引用不当会破坏阅读体验。

### 图表引用规则

| 规则 | 示例 |
|------|------|
| 图/表编号按章节编排 | "图2-3"表示第2章第3幅图 |
| 正文中引用用"如……所示" | "如图3-1所示，{{描述}}" |
| 引用放在描述句的句首或句中 | ✅ "系统的架构如图4-1所示。" |
| 不要将引用作为独立句子 | ❌ "图4-1展示了系统架构。" → 应融入上下文 |
| 表引用同样规则 | "各项指标对比见表6-2。" |
| 引用当前章图表无需章号 | "如图1所示（本章第1图）" |

### 图表标题规范

```
图标题置于图的下方：  "图3-2  {{模块名}}的内部结构示意图"
表标题置于表的上方：  "表5-1  各方法在{{数据集}}上的性能对比"
```

### 常见错误

```
❌ "下面的图展示了……"     → 用编号引用，不用方位词
❌ "请看下图"              → 同上
❌ "从上图可以看出"         → 用"由图X可知"
❌ 文中引用了图但没贴图     → 正文和图缺一不可
```

## 六、中英文混排规范

现代中文学术论文不可避免地涉及英文术语、缩写和文献引用，混排需要遵循统一规范。

### 术语首次出现

```
规则：中文术语（English Term, 缩写）
示例："本文采用卷积神经网络（Convolutional Neural Network, CNN）进行特征提取。"
     后续可直接使用"CNN"，无需重复全称。
```

### 标点符号

```
✅ 中文正文用中文标点（，。；：" "『』）
✅ 英文术语、公式、代码块中用英文标点（,.;:""''）
❌ 不要在中文正文中出现英文逗号和句号（除非整句英文）
✅ 中英文混排时，英文单词前后各留一个半角空格：
   "使用 Python 语言开发" ✓
   "使用Python语言开发"  ✗
```

### 大小写规范

```
✅ 专有名词保留原样大小写："JavaScript""MySQL""GitHub"
✅ 论文标题中英文术语可首字母大写（中文语境下）
✅ 正文中的普通英文术语用小写："该模块采用微服务（microservice）架构"
```

### 文献引用中的中英文

```
✅ 中文文献按 GB/T 7714 格式
✅ 英文文献按相应格式（APA/MLA/Chicago 等）
✅ 混排参考文献按首字母或拼音统一排序（看期刊要求）
```

> ⚠️ 以上所有示例均基于虚构的通用学术场景，不包含真实论文标题、系统名、人名或机构名。具体投稿时请以目标期刊/会议的**作者指南（Author Guidelines）**为准。
