# 摘要与正文映射验证 / Abstract-Body Mapping Verification

> 摘要是论文的"缩微版"。摘要中的每一个事实性声明，都必须在正文中找到对应的证据支持。本文件提供系统化的映射验证方法。

---

## 一、核心理念

摘要-正文不一致是论文投稿中常见的"致命伤"——审稿人往往首先对比摘要与正文中的数据。不一致会直接损害论文的可信度。

**原则**：摘要 ≠ 正文的简单截取，而是正文的精确映射。

---

## 二、映射验证表模板

### 2.1 逐条映射表格式

| # | 摘要中的声称 | 声称类型 | 正文对应位置 | 正文中的证据 | 验证结果 |
|---|-------------|---------|-------------|-------------|---------|
| 1 | "样本量为N=328家企业" | 数据 | 第3.1节 | "共发放问卷500份，回收有效问卷328份" | ✅ 已验证 |
| 2 | ... | ... | ... | ... | ... |

### 2.2 验证结果标记

| 标记 | 含义 | 后续操作 |
|------|------|---------|
| ✅ 已验证 | 摘要声称与正文完全一致 | 无需操作 |
| ⚠️ 部分匹配 | 有对应但数字/表述有偏差 | 统一表述 |
| ❌ 无匹配 | 摘要声称在正文中找不到对应 | 补充正文或删除摘要声称 |
| 🔴 矛盾 | 摘要与正文的数字/结论相反 | 核实原始数据，统一修正 |

## 三、常见不一致类型

### 3.1 样本量/数据量不一致

```
摘要："本研究收集了500份有效问卷"
正文（3.1节）："剔除无效问卷后，共获得328份有效问卷"

问题：摘要夸大了样本量（可能是初稿数据未更新）
修复：将摘要修正为328
```

### 3.2 统计结果不一致

```
摘要："两组差异显著（p<0.01）"
正文（4.2节）表格："p=0.083"

问题：p>0.05并不显著，摘要误写为显著
修复：核实统计结果，修正摘要或正文
```

### 3.3 百分比/数值不一致

```
摘要："效率提升了45.2%"
正文（5.1节）："处理时间从2.3秒降至1.1秒"（实际提升52.2%）

问题：计算方式不同或数据源不同
修复：统一计算口径
```

### 3.4 摘要提到了正文未分析的变量

```
摘要："...并分析了企业规模、行业类型和所有制结构的调节效应"
正文：只分析了企业规模的调节效应，未提及行业类型和所有制结构

问题：摘要超出了正文覆盖范围（"摘要领先正文"）
修复：从摘要中删除正文未覆盖的内容，或在正文中补充分析
```

### 3.5 结论强度不一致

```
摘要："本研究证明了数字化转型显著提升企业绩效"
正文（讨论部分）："相关性分析显示数字化转型与绩效提升存在
关联，但受制于横截面数据，无法建立因果关系"

问题：摘要用"证明"暗示因果关系，但正文讨论部分明确指出
     无法建立因果推断
修复：摘要改为"本研究发现数字化转型与企业绩效提升存在显著正相关"
```

## 四、映射验证工作流

### 4.1 四步验证流程

```
步骤1：提取
    └─ 从摘要中提取所有"可验证声明"
       （数字、统计结果、发现陈述、结论主张）

步骤2：查找
    └─ 在正文中查找每个声明对应的段落
       关注：方法部分（样本、数据）、结果部分（数字、统计）、
             讨论部分（解释）、结论部分（意义）

步骤3：标记
    └─ 对每个声明标记验证状态
       ✅已验证 / ⚠️部分匹配 / ❌无匹配 / 🔴矛盾

步骤4：修复
    └─ 摘要领先正文 → 正文需补充
       正文领先摘要 → 摘要需更新
       矛盾 → 核实后统一
```

### 4.2 可验证声明识别指南

以下类型的摘要内容必须能在正文中找到对应：

| 声明类型 | 示例 | 正文查找位置 |
|---------|------|-------------|
| 样本量 | "N=328" | 方法-参与者/样本 |
| 百分比 | "提升23.5%" | 结果-量化指标 |
| 统计值 | "p<0.01" | 结果-统计分析 |
| 方法名称 | "采用SEM分析" | 方法-数据分析 |
| 软件工具 | "使用SPSS 26.0" | 方法-工具 |
| 数据集 | "CIFAR-10" | 方法-数据 |
| 比较结论 | "优于基准方法" | 结果-对比分析 |
| 因果声明 | "证明了X导致Y" | 讨论-因果推断 |

## 五、虚构映射验证示例

### 5.1 摘要（虚构）

```
本文提出了一种基于深度迁移学习的轴承故障诊断方法（DTL-BFD），
在凯斯西储大学（CWRU）轴承数据集上进行了验证。实验结果表明，
在仅有10%目标工况标记样本的条件下，DTL-BFD的诊断准确率达到
96.7%，较微调CNN方法提升6.2个百分点。在跨工况迁移任务中，该
方法在6组迁移任务上的平均准确率为92.3%，优于4种基线方法。
消融实验表明，多核最大均值差异（MK-MMD）损失项对性能提升
贡献最大。该方法在仅有少量标记样本的工业场景中具有应用价值。
```

### 5.2 映射验证表

| # | 摘要声明 | 类型 | 正文位置 | 正文证据 | 结果 |
|---|---------|------|---------|---------|------|
| 1 | "10%目标工况标记样本" | 数据 | 4.1节实验设置 | "目标域随机选取10%样本作为标记数据（每类约25个样本）" | ✅ 已验证 |
| 2 | "诊断准确率96.7%" | 数据 | 4.2节表2 | 表2：DTL-BFD最佳准确率=96.7%±0.4% | ✅ 已验证 |
| 3 | "较微调CNN提升6.2pp" | 比较 | 4.2节表2 | CNN微调准确率=90.5%，差值为6.2pp | ✅ 已验证 |
| 4 | "6组迁移任务" | 数据 | 4.3节 | "设计了6组跨工况迁移任务" | ✅ 已验证 |
| 5 | "平均准确率92.3%" | 数据 | 4.3节表3 | 表3：6组任务平均=92.3% | ✅ 已验证 |
| 6 | "优于4种基线方法" | 比较 | 4.3节 | 对比了DDC, DAN, CDAN, MCD共4种方法 | ✅ 已验证 |
| 7 | "MK-MMD贡献最大" | 发现 | 4.4节消融 | 表4：移除MK-MMD后性能下降最多（-5.7pp） | ✅ 已验证 |
| 8 | "少量标记样本的应用价值" | 结论 | 5.2节 | 讨论了在少样本工业场景的适用性 | ⚠️ 部分匹配（讨论较笼统） |

### 5.3 不一致示例

```
摘要："该方法在6组迁移任务上的平均准确率为92.3%"
正文表3：6组任务准确率分别为 88.1%, 91.4%, 93.7%, 90.2%, 94.8%, 89.6%
          实际平均 = (88.1+91.4+93.7+90.2+94.8+89.6)/6 = 91.3%

🔴 矛盾！摘要写的92.3%，实际平均值为91.3%。
修复：核实原始数据后统一数字。
```

## 六、修复策略

### 6.1 摘要领先正文（摘要说了正文没说的）

```
场景：摘要提到了"所有制结构的调节效应"，正文没有相应分析
策略A（推荐）：从摘要中删除该声明
策略B：在正文中补充相应分析（仅在数据支持时）
```

### 6.2 正文领先摘要（正文有重要发现摘要遗漏了）

```
场景：正文发现了"网络密度负向调节创新投入与创新绩效的关系"
      但摘要只说了"创新投入正向影响创新绩效"
策略：在摘要的"结果"部分补充调节效应的发现
```

### 6.3 数据矛盾

```
场景：摘要和正文对同一指标的数字不同
策略：
  1. 核实原始数据
  2. 追溯不一致原因（计算方式？初稿版本？四舍五入？）
  3. 统一修改为正确值
  4. 确保全文（摘要+正文+图表）一致
```

## 七、批量验证检查脚本逻辑（手动执行参考）

```
FOR 摘要中的每个句子:
    IF 句子包含数字 OR 统计结果 OR 比较声明:
        提取声明 → 搜索正文 → 对比 → 标记结果
    
FOR 正文"结果"部分的每个关键发现:
    IF 发现未出现在摘要中:
        标记为"正文领先摘要"

OUTPUT: 映射验证报告
  ├── 总声明数: N
  ├── 已验证: N
  ├── 部分匹配: N （列出）
  ├── 无匹配: N （列出）
  ├── 矛盾: N （列出）
  └── 正文领先摘要: N （列出）
```
