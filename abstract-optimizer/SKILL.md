# abstract-optimizer — 摘要优化与审查

## 定位说明

本技能是**摘要质量审查与优化工具箱**，不负责生成摘要或评分——那是 `abstract-summarizer`（生成）和 `abstract-logic-writer`（评分）的职责。本技能关注的是：摘要写好之后，它是否规范、中英文是否对称、数据是否与正文一致、关键词是否科学选配。

### 与已有 SkillHub 技能的关系

| 已有技能 | 职责 | 版本 | 本技能与之关系 |
|---------|------|------|-------------|
| `abstract-summarizer` | 长文→250字摘要生成 | v0.1.0 | 互补：它生成初稿，本技能审查优化 |
| `abstract-trimmer` | 压缩摘要到字数限制 | v0.1.0 | 互补：它做长度裁剪，本技能保证裁剪后质量 |
| `abstract-logic-writer` | CS专用符号规则评分 | v0.1.1 | 互补：它评分逻辑，本技能审查对称性+规范 |
| `conference-abstract-adaptor` | 适配会议格式要求 | v0.1.0 | 互补：它做格式适配，本技能做内容审查 |

**核心差异**：以上4个技能都围绕"生成/裁剪/评分/适配"摘要内容，无人覆盖**中英双语对称性审查、学科特定模板、摘要-正文映射验证、摘要审查清单**四大维度。本技能填补这一空白。

## 使用场景

- 论文投稿前，检查中英文摘要是否对称、规范
- 摘要与正文初稿已完成，验证两者数据是否一致
- 按照目标学科/期刊的摘要模板重构摘要结构
- 优化关键词选择，使中英文关键词概念对应、检索覆盖最大化
- 逐条对照30项审查清单，确保摘要全面合规

## 工作流程

```
用户提交摘要（中文 + 英文 + 可选正文）
        │
        ├─→ 步骤1: 中英对称性审查（references/cn-en-abstract-spec.md）
        │   核对要素对应、数据一致、时态一致、比例合理
        │
        ├─→ 步骤2: 学科模板匹配（references/discipline-templates.md）
        │   识别论文学科，推荐对应摘要模板
        │
        ├─→ 步骤3: 摘要-正文映射验证（references/abstract-body-mapping.md）
        │   提取摘要中所有事实断言，在正文中逐条验证
        │
        ├─→ 步骤4: 审查清单评分（references/abstract-review-checklist.md）
        │   6大类30项逐条审查，输出通过率
        │
        ├─→ 步骤5: 关键词优化（references/keyword-optimization.md）
        │   审查中英文关键词的选取质量与对称性
        │
        └─→ 输出: 审查报告 + 优化建议 + 修改后摘要
```

## 质量标准与红线

### 四维质量评分

| 维度 | 定义 | 优秀(3分) | 达标(2分) | 不及格(0分) |
|------|------|-----------|-----------|------------|
| 中英对称性 | 要素对称/数据一致/时态一致/比例合理 | 30项清单全绿 | ≥85%通过率 | <70%通过率 |
| 结构规范性 | 四要素/五要素框架完整 | 完整匹配模板 | 1-2项偏离 | 结构不清晰 |
| 数据一致性 | 摘要断言在正文中可定位验证 | 100%可映射 | ≥80%可映射 | <60%可映射 |
| 关键词科学性 | 选取策略+中英对称+检索覆盖 | 3维全部达标 | 2维达标 | 1维以下 |

加权总分 = Σ(维度分) / (4×3) × 100
优质线：≥75分 ｜ 合格线：≥50分 ｜ 重做线：<50分

### 质量红线

- 🔴 绝不改写非用户提供的现有摘要——本技能是审查器，非生成器
- 🔴 绝不到正文中找摘要中没有的描述——映射验证必须基于用户提供的正文
- 🔴 不编造学科模板——仅使用已掌握的四学科模板，超出时搜索后提供

## 联网规则

1. 当需要期刊格式要求时，搜索当用户不确定目标期刊的摘要格式时，该期刊的Author Guidelines
2. 当需要学科最新规范时，搜索当学科非工程/管理/医学/CS四类时，该学科摘要规范
3. 任何联网获取的信息必须注明来源
4. 联网结果仅供辅助决策，不替代用户自身判断
5. 如果联网失败，基于已有知识和模板继续工作并告知用户
6. 超出知识范围时优先联网搜索而非猜测
