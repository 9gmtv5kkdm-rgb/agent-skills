## 方法论：四步包管理

### Step 1：自检（Check）
遍历11个技能目录，检查 `~/.qclaw/skills/{name}/SKILL.md` 是否存在。
统计已装/缺失数量。

### Step 2：安装（Install）
对缺失技能执行 `skillhub_install {name}`，按论文阶段顺序安装（先文献→后写作→再审查→最后投稿）。
每装完1个报进度。

### Step 3：验证（Verify）
全部安装后抽取3个技能做快速功能验证：
- 查重卫士：输入「帮我看看这篇论文的重复率」→应正确路由
- 论文审校：输入「帮我审查这篇论文」→应正确路由
- 投稿指南：输入「帮我写Cover Letter」→应正确路由

### Step 4：引导（Guide）
展示论文写作七阶段工作流，告知用户在哪个阶段用哪个技能。

## 论文七阶段工作流

```
①文献调研 → 文献研究 + 查重卫士
②大纲框架 → 写作词库
③数据图表 → 学术制图 + 架构绘图
④初稿撰写 → 写作词库 + 摘要精修
⑤修改审查 → 论文审校
⑥排版定稿 → 论文排版 + 引文规范 + 版本管家
⑦投稿发表 → 投稿指南
```

## 自动安装规则

> 首次触发时自动执行，后续触发可在 S2 场景手动触发。

1. 遍历11个技能目录，检查本地是否存在 SKILL.md
2. 列出已装和缺失的技能清单
3. 询问用户「检测到X个技能缺失，是否一键安装？」（确认一次）
4. 得到确认后按阶段顺序逐个安装
5. 全部完成后展示论文七阶段工作流
6. 告知用户：每个子技能内部也有「学术技能网·自动补全」——装任意一个都能联动补全全套

## 自进化体系

### L1 架构自知
- 核心假设：论文写作是阶段化流程，按阶段路由到对应子技能
- 易变点：新论文阶段可能随学术规范变化而出现
- 组件：4场景路由 + 四步包管理 + 七阶段工作流 + 11依赖技能

### L2 循环验证
- 每次触发时执行自检，确认11个依赖技能路径存在
- 统计路由命中率

### L3 信号检测
- 新增学术技能→更新依赖清单→更新本技能
- 依赖技能版本升级→提示用户

### L5 互斥声明
- 本技能是路由入口，不执行具体论文操作，与所有11个子技能无边界冲突
- 与 skill-maker-pro 互补：元技能只管制作技能，学术全能包只管安装和使用

### 🔒 不可变核心
1. 依赖技能清单在变更前必须经用户确认
2. 安装操作必须告知用户正在安装哪个技能（不静默安装）
3. 首次自检结果必须展示给用户
